⚡ On-Demand Session Flow
1
User describes test → Agent reads DynamoDB SERVICE#{team}#{env} → generates plan
2
User selects service chip + TC name → Save Test Case → Lambda saves to Aurora + embeds vector
3
User opens TestCasePage → clicks Automate → popup opens instantly with progress bar
4
start_session: reads SSM config → RunTask (dedicated Fargate task)
5
Poll DescribeTasks (~60s) → RUNNING → public IP via ENI → TCP poll port 3000 until ready
6
Pre-warm: MCP connect → browser_navigate about:blank → Chromium open in noVNC
7
automate: LLM drives Playwright via MCP → captures playwright_calls per step → streams results
8
User saves steps → Save Steps persists playwright_calls to Aurora via Lambda
9
Run Test: replay mode → raw ClientSession.call_tool() (no LLM) → StopTask → PASS/FAIL recorded
