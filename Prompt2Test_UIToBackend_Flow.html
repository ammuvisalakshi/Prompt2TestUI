UI to Backend — Code Flow

A code-level walkthrough of every call from the React frontend to AgentCore, ECS, and back — tracing each file, function, and payload through the system.

1

Plan Mode Flow

User describes a test → React sends a signed request to AgentCore → Claude reasons and returns a structured JSON plan → UI renders it as numbered steps. No browser is started and no MCP calls happen here.

1
AgentPage.tsx — send()

User types a prompt and submits

The message is appended to the chat. send() builds conversationHistory from all prior messages so the agent has memory across turns, then calls callAgent() with mode: "plan".

const history = messages .map(m => `${m.role === 'user' ? 'User' : 'Agent'}: ${m.text}`) .join('\n') callAgent({ inputText: text, mode: 'plan', sessionId, conversationHistory: history }, sessionId)
2
AgentPage.tsx — callAgent()

Cognito credentials fetched → request signed → sent to AgentCore

fetchAuthSession() retrieves temporary AWS credentials from Cognito. A BedrockAgentCoreClient is constructed with those credentials, then InvokeAgentRuntimeCommand is sent — the SDK automatically SigV4-signs it. The payload is JSON-encoded as Uint8Array.

const session = await fetchAuthSession() // Cognito temp creds const client = new BedrockAgentCoreClient({ region, credentials: session.credentials }) const cmd = new InvokeAgentRuntimeCommand({ agentRuntimeArn: AGENT_RUNTIME_ARN, runtimeSessionId: sessionId, payload: new TextEncoder().encode(JSON.stringify(payload)), }) const response = await client.send(cmd)
3
main.py — POST /invocations

AgentCore receives the request

FastAPI reads the body, extracts sessionId, mode, and inputText. Seeing mode: "plan", it calls runner.plan().

mode = body.get("mode", "plan") if mode == "plan": result = runner.plan( prompt=body.get("inputText"), session_id=session_id, conversation_history=body.get("conversationHistory", ""), ) return JSONResponse({ "sessionId": ..., "mode": "plan", "plan": result["plan"] })
4
agent_runner.py — AgentRunner.plan()

Strands Agent invokes Claude on Bedrock

A Strands Agent is built with SYSTEM_PROMPT and two lightweight tools: resolve_config (returns {{KEY}} placeholder) and resolve_secret (returns ***name***). Conversation history is prepended to the prompt so Claude remembers prior turns. Claude returns a JSON plan — or a clarifying question if it needs more info.

agent = Agent( model=_build_model(), # claude-sonnet-4-5 via Bedrock system_prompt=SYSTEM_PROMPT, tools=_build_tools_phase1(), # resolve_config, resolve_secret ) full_prompt = f"Conversation so far:\n{history}\n\nLatest: {prompt}" response = agent(full_prompt) plan = self._parse_plan(str(response)) # strips markdown, extracts JSON
5
AgentPage.tsx — callAgent() response reader

Streaming response parsed → plan rendered in UI

The response is a ReadableStream. The UI reads it chunk by chunk, splitting on newlines to parse NDJSON events as they arrive. The final JSON is parsed into a Plan object, stored in state, and rendered as numbered steps in the right panel. The confirmationMessage is shown in chat. The agent asks: "Would you like me to execute this?"

// Plan shape returned by Claude: { "confirmationMessage": "Got it! We'll navigate to amazon.com...", "summary": "Search for iPhone 17 on Amazon", "steps": [ { "stepNumber": 1, "type": "navigate", "tool": "playwright", "action": "Go to amazon.com", "detail": "Navigate to https://amazon.com" } ], "estimatedTokens": 320, "mcpCalls": 4 }

Automate Mode — Call 1: start_session

User types "yes" / "run". The frontend detects this, opens a blank popup window synchronously (before any await to avoid browser popup blockers), then calls start_session to claim a warm ECS task and get the live browser URL.

1
AgentPage.tsx — send()

Confirmation detected → popup opened synchronously

The UI checks if a plan exists and the text matches /^(yes|run|execute|go|ok|sure|start|do it)/i. window.open() is called before any await — this is critical. Browsers only allow popups triggered directly by user gesture; any async gap would cause it to be blocked.

if (plan && /^(yes|run|execute|go|ok|sure|start|do it)/i.test(text)) { // MUST be before any await — opens within user gesture const popup = window.open('about:blank', 'novnc-popup', 'width=1280,height=820') // Now we can await const sessionRaw = await callAgent({ mode: 'start_session', ... })
2
main.py → agent_runner.py — start_session()

AgentCore claims the warm ECS task

ECSSession._start() reads cluster name and task config from SSM Parameter Store, then calls ECS ListTasks to find the already-running warm task. It grabs the task's public IP and constructs the noVNC and MCP URLs. The task is not started — it was already running in the warm pool. No wait. Returns immediately.

from agent.ecs_session import ECSSession ecs = ECSSession(region=self.region) ecs._start() # reads SSM → ListTasks → gets public IP return { "novnc_url": ecs.novnc_url, # http://3.91.x.x:6080/vnc.html "mcp_endpoint": ecs.mcp_endpoint, # http://3.91.x.x:3000 "task_arn": ecs.task_arn, "cluster": ecs.cluster, }
3
AgentPage.tsx

noVNC URL received → popup navigated to live browser

The response is parsed. The already-open blank popup is navigated to the noVNC URL with autoconnect=true&resize=scale. The browser connects directly to the ECS task at IP:6080 — no proxy, no CloudFront. The user can now watch the live browser.

const sessionResult = JSON.parse(sessionRaw) const novncUrl = sessionResult.novnc_url if (popup) popup.location.href = `${novncUrl}?autoconnect=true&resize=scale` // Popup now shows live Chromium browser running on the ECS task
3

Automate Mode — Call 2: automate

With the live browser already visible in the popup, the frontend sends the plan to automate. AgentCore connects to playwright-mcp via MCP/SSE and runs a Claude agent loop — one tool call per plan step. The ECS task is stopped when the test finishes.

1
AgentPage.tsx

automate payload sent with plan + task info

callAgent() is called with mode: "automate", the full plan object, and the task_arn, cluster, and mcp_endpoint returned by start_session. This lets the backend reuse the already-running task without starting a new one.

const raw = await callAgent({ inputText: text, mode: 'automate', plan, // the plan JSON from step 1 sessionId, task_arn: sessionResult.task_arn, cluster: sessionResult.cluster, mcp_endpoint: sessionResult.mcp_endpoint, }, sessionId)
2
main.py → agent_runner.py — automate_stream()

AgentCore connects to playwright-mcp via SSE

Seeing task_arn + cluster + mcp_endpoint in the payload, automate_stream() takes the pre-started task path. It builds an MCP client pointing to mcp_endpoint/sse and overrides the Host header to localhost:3000 — playwright-mcp validates the Host for CSRF protection and only accepts localhost.

def _build_mcp_client(endpoint): sse_url = endpoint.rstrip("/") + "/sse" port = urlparse(endpoint).port or 3000 host_header = f"localhost:{port}" # bypasses playwright-mcp CSRF check return MCPClient(lambda: sse_client(sse_url, headers={ "Host": host_header }))
3
agent_runner.py — automate_stream()

Strands Agent loop runs — one MCP tool call per plan step

A Strands Agent is built with the MCP tools from playwright-mcp (playwright_navigate, playwright_click, playwright_fill, playwright_get_visible_text, playwright_screenshot, etc.). Claude reads the full steps text and executes each one in order, calling the tool, observing the result, and deciding pass/fail before moving to the next step. The user watches every action happen live in the noVNC popup.

with _build_mcp_client(mcp_endpoint) as mcp: tools = mcp.list_tools_sync() # playwright_navigate, _click, _fill… agent = Agent( model=_build_model(), system_prompt=AUTOMATE_SYSTEM_PROMPT, tools=tools, ) response = agent(prompt) # Claude loops until all steps done
4
agent_runner.py — automate_stream()

Result yielded → ECS task stopped

After the agent loop completes, _parse_plan() extracts the result JSON from Claude's response. A complete NDJSON event is yielded. Then ECSSession._stop() calls ECS StopTask. The ECS Service detects the stopped task and automatically starts a replacement — restoring the warm pool for the next test run.

finally: ecs = ECSSession(region=self.region) ecs.task_arn = task_arn ecs.cluster = cluster ecs._stop() # StopTask → ECS Service auto-replaces yield json.dumps({ "event": "complete", "result": result }) + "\n"
5
AgentPage.tsx

Result displayed in chat

The streaming response is read until the complete event arrives. result.passed determines whether to show ✅ or ❌. The summary and step details are shown in the chat panel.

const result = JSON.parse(raw) const passed = result.result?.passed ?? result.passed // Result shape from Claude: { "passed": true, "summary": "All 4 steps passed", "steps": [{ "stepNumber": 1, "status": "passed", "detail": "Navigated to amazon.com" }] }

End-to-End Call Map

Three distinct calls to AgentCore, each with a different mode. The frontend orchestrates them in sequence.

🔐

Auth

Every call is authenticated. fetchAuthSession() gets Cognito temp credentials. The SDK SigV4-signs all requests automatically.

🧠

State

Session ID is a stable UUID per page load. Conversation history is passed as a string on every plan call — AgentCore itself is stateless.

🪟

Popup timing

window.open() must be called synchronously inside the click handler. Any await before it causes browsers to block the popup.

♻️

Warm pool

start_session claims an already-running ECS task — no cold start wait. StopTask at the end triggers ECS to auto-replace it for the next run.
