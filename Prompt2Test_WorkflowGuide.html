Technical Guide

How Prompt2Test Works

A complete walkthrough of the platform architecture, the Plan Mode workflow, and the Automate Mode workflow — from user prompt to live browser execution.

1

Overall Platform Architecture

Prompt2Test is a fully serverless AI-powered test automation platform built on AWS. Users describe a test in plain English. The AI agent plans and executes it in a real browser, while the user watches live. No test code needs to be written.

🖥️

Frontend

React app on AWS Amplify Hosting. Cognito handles login. Users interact via a chat-style interface.

🤖

AI Agent

Python Strands agent on AgentCore. Uses Claude Sonnet 4.5 on Bedrock for all AI reasoning.

🌐

Browser Automation

playwright-mcp on ECS Fargate. Warm pool keeps one task running. Real Chromium browser, viewed live via noVNC popup at direct IP:6080.

🔄

CI/CD

Two CodePipeline pipelines (agent + mcp images) and Amplify built-in CI/CD for the frontend.

2

Plan Mode Workflow

When the user types a test description and clicks Plan, the system generates a structured, step-by-step test plan using Claude. No browser is opened — this is pure AI reasoning. The result is a human-readable plan the user can review and refine before execution.

StepServiceWhat happensBrowser?MCP call?
1 User types prompt Amplify Frontend User describes the test in plain English and clicks Plan ❌❌
2 InvokeAgentRuntime AgentCore Frontend calls AgentCore via @aws-sdk/client-bedrock-agent-runtime with { mode:"plan", inputText:"..." }, signed with Cognito SigV4 credentials ❌❌
3 agent_runner.plan() AgentCore Python Python agent formats the prompt and sends it to Claude via Strands SDK ❌❌
4 Claude reasons Amazon Bedrock Claude Sonnet 4.5 reads the prompt and generates a structured JSON plan with steps, tools, and estimates ❌❌
5 Plan displayed Amplify Frontend React renders the plan as readable steps. User reviews and clicks Run to execute ❌❌
3

Automate Mode Workflow

When the user clicks Run, the system uses a 2-call flow: first start_session claims a warm ECS task and returns a noVNC URL immediately (the live browser popup opens right away), then automate connects to playwright-mcp via direct IP and executes the plan step by step. The ECS task is stopped when the test finishes — ECS Service auto-replaces it.

StepServiceWhat happensBrowser?MCP call?
1 User clicks Run Amplify Frontend Frontend calls mode:"start_session". AgentCore reads SSM params, claims the warm ECS task via ListTasks, returns novnc_url + task_arn + mcp_endpoint ⏳ starting❌
2 noVNC popup opens Amplify Frontend Frontend sets popup.location.href = novnc_url (direct IP:6080). Live browser view is immediately visible while the automate call runs in parallel ✅ popup❌
3 MCP client connects AgentCore → playwright-mcp Frontend calls mode:"automate" with the plan and task info. Python agent opens SSE connection to playwright-mcp at direct IP:3000. Host header set to localhost:3000 to bypass CSRF check ✅✅ connect
4 Agent loop begins Bedrock Claude + Strands Claude reads the plan and decides which MCP tool to call for step 1 ✅✅
5 Browser action playwright-mcp → Chromium playwright-mcp executes the tool (navigate, click, fill, assert, screenshot) in real Chromium on Xvfb ✅ action✅ per step
6 Live view streams x11vnc → websockify → noVNC popup x11vnc captures the Xvfb screen, websockify converts to WebSocket, noVNC serves it at direct IP:6080. User watches live in the popup window. ✅ visible❌
7 Result → Claude Bedrock Claude playwright-mcp returns the tool result. Claude decides: pass and move to next step, retry, or mark as failed ✅✅
8 Loop repeats All Steps 4–7 repeat for every step in the plan until all steps complete or a step fails ✅✅
9 Final result + cleanup AgentCore → ECS → Amplify Frontend AgentCore returns { passed:true/false, steps:[...] }. StopTask is called on the ECS task. ECS Service auto-replaces the warm task. UI shows pass/fail per step. ✅ closes❌
🔄

Agent Loop

Claude + Strands SDK loop: think → call tool → observe result → repeat. One MCP call per plan step.

🌐

Real Browser

Actual Chromium running headed on a virtual display (Xvfb). Not a simulation — real DOM, real network requests.

📺

Live View

x11vnc → websockify → noVNC popup at direct IP:6080. Every click and scroll is visible in real time.

✅

Pass / Fail

Each step is independently assessed. Claude can retry before marking a step failed. ECS task stopped on completion.
